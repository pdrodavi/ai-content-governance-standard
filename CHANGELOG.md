# Changelog

All notable changes to this project will be documented in this file.

This project follows a **public, versioned standard model**, where changes are explicit, traceable and reviewable.

---

## [1.0.0] — Public Initial Release
### Added
- Public AI Content Governance Standard
- Normative core chapters (`/docs`)
- Explicit separation between decision-making and LLM generation
- Legal threat model focused on content delivery
- Policy-as-code approach
- Decision versioning model (`policyVersion`, `policyHash`, `effectiveDate`)
- OpenAPI specification for normative decisions
- Executable policies using OPA / Rego
- Full audit and evidence model
- AI Impact Assessment (AIA / DPIA)
- Jurisdictional annexes (Brazil, European Union, United States)
- External audit checklist
- Regulatory incident response runbook
- Architecture and operational flow diagrams

---

---

### Versioning Policy

- **MAJOR** — normative or architectural breaking changes  
- **MINOR** — additive clarifications or new optional components  
- **PATCH** — editorial fixes, clarifications, or non-normative improvements  

---

Governance is iterative by design.
